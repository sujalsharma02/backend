# **App Name**: Topic Explorer API

## Core Features:

- Topic Retrieval: Retrieve all topics from the data source.
- Search Topics: Filter topics by name using a case-insensitive search of the `name` field.
- Sort Topics: Sort topics by name in ascending order based on the 'sort=name' query parameter.
- Error Handling: Handle invalid queries and file errors, returning appropriate HTTP status codes (400, 500).

## Style Guidelines:

- Primary color: Cool blue (#468B97), evoking trustworthiness and clarity, aligning with the data-focused nature of an API.
- Background color: Very light gray (#F0F4F5), for a clean and neutral backdrop that puts focus on the API's data responses.
- Accent color: Soft green (#A3D1A8), to indicate success states, available parameters, or other interactive elements within documentation.
- Body and headline font: 'Inter', sans-serif, to ensure readability and a modern, clean aesthetic for API documentation.
- Code font: 'Source Code Pro' for displaying API endpoints and JSON responses.
- Clear and structured layout for API documentation to ensure ease of use.
- Subtle transitions or highlights when interacting with API documentation (e.g., when a user performs a search or sorts topics).
